# 字幕翻译工作室

这是一个本地网页软件：上传视频或填写本机视频路径后，自动完成日语语音检测、喘息/呻吟甄别、日文识别、中文翻译、质量门禁和字幕封装。既可使用本地 Whisper 与 Ollama，也保留 OpenAI 兼容接口，可从接口读取模型列表并选择模型。

## 快速启动

1. 安装 Python 依赖：`python -m pip install -r requirements.txt`
2. 确认 `ffmpeg` 和 `ffprobe` 已加入 PATH。
3. 双击 `启动字幕工作室.bat`，浏览器会自动打开 `http://127.0.0.1:8765`。
4. 选择浏览器上传，或直接填写本机视频绝对路径。
5. 选择识别、复核、翻译模型以及质量档位后创建任务。

任务按单 GPU 队列依次执行。任务页会显示阶段日志，并提供中文字幕、日文字幕、中日双语字幕、结构化 JSON、质量报告以及软／硬字幕视频的下载入口。软字幕可在播放器中开关且封装很快；硬字幕会重编码画面，耗时更长，但兼容不支持外挂字幕的设备。

## 模型入口

- 本地识别：Whisper `small`、`medium`、`large-v3` 等已缓存模型。
- 本地翻译：Ollama 模型，可从本机 `/api/tags` 自动读取。
- OpenAI 兼容识别：调用 `/v1/audio/transcriptions`；服务必须返回带片段时间戳的 `verbose_json.segments`，否则无法还原视频时间轴。
- OpenAI 兼容翻译：调用 `/v1/chat/completions`，并通过 `/v1/models` 获取可选模型。
- API Key 只保存在当前任务进程内，写入任务状态文件前会被清空。

## 三档质量机制

- 高精度：更严格地排除喘息、呻吟和静音幻听，适合对白较少的片段，但可能漏掉极轻对白。
- 均衡：默认档。维持喘息过滤，同时对较长对白空洞进行二次搜索。
- 高召回：降低首次门槛并增强长空档恢复，适合对白被喘息覆盖的片段；建议结合“高置信校对版”复查。

固定质量门禁包括：双模型复核、长空档恢复候选二次验证、字幕最短显示时间、碎片合并、重叠修复、占位符发布清理、否定/停止类语义审计，以及中文字幕句号移除。观看版不会把未确认的喘息强行翻译成剧情台词；校对版保留更适合人工复核的结果。

## 输出目录

运行数据位于 `studio_data/`：

- `uploads/`：浏览器上传的原视频。
- `jobs/<任务 ID>/output/`：字幕、软字幕视频和质量报告。
- `jobs/<任务 ID>/status.json`：可恢复的任务状态，不含 API Key。

## 命令行启动

不希望自动打开浏览器时，可运行：

```powershell
python -m uvicorn studio.main:app --host 127.0.0.1 --port 8765
```

## 领域增强流程

这套流程用于处理包含长静音、喘息、呻吟、水声和少量真实对白的视频。目标是优先降低“非语言声音被识别成台词”的假阳性，同时保留喘息中仍然存在的真实日语。

## 流程

1. `vad_scan.py`：使用 WebRTC VAD 找出人声活动候选；严格模式仍会包含喘息。
2. `audio_event_gate.py`：使用 AudioSet AST 模型区分 Speech 与 Breathing、Pant、Wheeze、Gasp、Moan、Water 等声音事件。
3. `asr_stage.py`：将候选片段压缩打包，使用 Whisper medium 识别并生成词级时间戳；过滤静音幻听、重复、低置信度和高压缩异常。
4. `large_review.py`：只对 medium 保留的对白使用 Whisper large-v3 复核，避免全片运行 large-v3 的巨大开销。
5. `studio/recall.py`：扫描较长无字幕区间，只接受声音门控与双模型一致性同时通过的二次召回结果。
6. `studio/translation.py`：逐句翻译，每次只允许返回一个目标句；前后文只用于理解。自动审计否定词、`やめて`、`待って`、`聞いてない` 和 `勘弁`。
7. `studio/quality.py` 与 `studio/subtitles.py`：合并碎片、修复重叠、执行发布清理，并生成字幕和软／硬字幕视频。

## 关键设计

- 纯喘息/呻吟不强行翻译；无法确认时标记 `【听不清】`。
- 翻译逐句执行，避免批量 JSON 中发生整条错位。
- 采用词级或语义级时间轴，不使用固定两秒切片。
- 日语原稿不确定时先保留不确定性，不让中文模型补写剧情。
- 对拒绝、否定和停止表达执行独立语义审计。

## 局限

- 这不是使用某一类题材数据重新训练的专用模型，而是预训练声音事件模型、双 ASR 和规则审计组成的领域增强管线，因此也可用于访谈、课程和普通影视素材。
- 精度优先会漏掉少量非常轻、被喘息完全覆盖的真实词语。
- OpenAI 兼容服务实现并不完全一致；远程 ASR 必须支持多段表单上传以及 `verbose_json` 片段时间戳。
- 软字幕是可开关的字幕轨，不会破坏原始画面；硬字幕会永久烧录在画面中。软件默认勾选软字幕，硬字幕按需开启。
