## 项目目标

提供一个可复用的本地网页入口，让用户上传视频或填写本机媒体路径后，自动完成语音活动检测、非语言声音甄别、日语识别、中文翻译、字幕质量门禁以及软／硬字幕视频生成。软件不绑定特定影片，可用于影视、访谈、课程及其他包含日语语音的视频。

## 需求与改进来源

机制依据实际字幕质量评价逐步固化，重点处理以下问题：

- 喘息、呻吟、水声和静音容易被误识别成对白。
- 过度追求精度会导致轻声对白召回下降。
- 较长无字幕区间需要二次检查，但不能无条件补写台词。
- 批量翻译可能错位，否定、拒绝和停止表达需要独立审计。
- 中文字幕不需要行末句号，字幕不能过短或互相重叠。
- 模型入口必须同时支持本地模型与 OpenAI 兼容接口。

## 当前架构

1. `vad_scan.py` 使用 WebRTC VAD 获取声音活动候选。
2. `audio_event_gate.py` 使用 AudioSet AST 区分语音与喘息、呻吟、水声等声音事件。
3. `asr_stage.py` 使用本地 Whisper 识别并过滤重复、静音幻听和低置信异常；远程识别由 `studio/remote_asr.py` 调用兼容接口。
4. `large_review.py` 使用第二个本地 Whisper 模型复核初筛对白。
5. `studio/recall.py` 对较长空白执行临界候选二次召回，并要求双模型一致。
6. `studio/translation.py` 通过本地 Ollama 或 OpenAI 兼容 Chat Completions 逐句翻译并审计高风险语义。
7. `studio/quality.py` 完成碎片合并、最短时长、时间轴重叠修复、占位符清理和中文句号移除。
8. `studio/subtitles.py` 输出中文、日文、双语 SRT、JSON、质量报告以及软／硬字幕视频。
9. `studio/main.py` 与 `studio/static/` 提供上传、参数设置、任务队列、日志和产物下载网页。

## 三档质量策略

- 高精度：提高声音事件门槛，优先降低非语言声音假阳性。
- 均衡：默认策略，兼顾精度与召回，并开启长空白二次检查。
- 高召回：降低临界人声门槛，适合轻声对白较多的素材，建议结合校对版复核。

## 模型与接口

### 本地模型

- Whisper 模型由 `C:\Users\28448\.cache\whisper` 管理。
- 迁移时确认已缓存：`small`、`medium`、`large-v3`。
- Ollama 模型由 Ollama 自身管理；迁移时确认可用：`qwen2.5:7b-instruct`。
- 上述系统级模型没有随旧工程缓存删除，确保新目录仍能直接使用本地模型。

### OpenAI 兼容接口

- 模型列表：`/v1/models`
- 语音识别：`/v1/audio/transcriptions`
- 翻译：`/v1/chat/completions`
- 远程 ASR 必须返回 `verbose_json.segments`，否则无法映射回原视频时间轴。
- API Key 只在当前进程中使用，保存任务状态时会清空。
- Ollama 与 OpenAI 兼容提供方分别保存 Base URL，避免把 `/api/tags` 错误发送到 OpenAI 地址。

## 维护说明

- 修改接口或质量机制后，应重新执行 `python -m compileall -q .`。
- 若安装了测试文件，可执行 `python -m unittest discover -s tests -v`。
- 长视频建议使用 `medium` 主识别、`large-v3` 复核和均衡策略。
- 显存不足时可将主识别改为 `small`，复核仍可保留 `large-v3`，因为复核只处理候选对白。
- 生成硬字幕会重编码整部视频；如无兼容性要求，优先使用软字幕。
