# 云端最高精度多模型识别

该模式面向 32GB 单卡云算例，目标是减少弱对白漏识别、背景音乐干扰误识别和单模型幻听。

## 模型链

1. `Qwen3-ASR-1.7B` 对低门槛语音候选做主识别，每批 4 个窗口。
2. 程序根据语音事件分数、音乐/噪声遮蔽、文本过短和重复输出自动选出低置信窗口。
3. `cohere-transcribe-03-2026` 每批 4 个窗口独立复核低置信结果。
4. Qwen 与 Cohere 的规范化文本相似度低于 0.72 时，才加载 `large-v3` 作为第三意见。
5. 最终文本由 `Qwen3-ForcedAligner-0.6B` 每批 8 个窗口执行日语或韩语词级时间轴对齐。

四个阶段有数据依赖，按顺序运行。每个模型位于独立进程或独立 Python 环境，阶段结束后 CUDA 上下文随进程退出，避免四个模型同时占用显存。各阶段内部使用批量并行推理。

## 当前云算例参数

- GPU：32GB RTX 4080 SUPER / vGPU 32GB
- CPU：12 vCPU
- 内存：62GB
- 建议数据盘：100GB；安装前至少需要 35GB 可用空间
- 默认模型目录：`/root/autodl-tmp/subtitle-models`
- 任务工作目录：`/root/subtitle-worker`

模型、虚拟环境和 SHA-256 清单全部放在数据盘。系统盘只保留原有工作程序。Qwen、ForcedAligner 和 Cohere 从 ModelScope 下载，避免运行任务时依赖 Hugging Face。

## 启用步骤

1. 等现有旧版任务正常结束。
2. 将 AutoDL 数据盘扩到建议容量并开机。
3. 重启本地字幕工作室，使版本升级生效。
4. 在“云 GPU 运算单元”中确认 SSH 配置和模型数据盘目录。
5. 点击“安装最高精度模型”。依赖和三套新权重会并行安装，随后生成 SHA-256 清单。
6. 识别提供方选择“云端最高精度多模型”，再创建任务。

原视频继续留在本机，仅上传 16kHz 单声道 FLAC 音轨。云端只返回识别 JSON，本机继续完成翻译、自动文本校正和字幕产物生成。

## 容错

- 模型目录可用空间不足 35GB 时拒绝安装，不占满系统盘。
- 已安装标记与三套模型配置都存在时跳过重复下载。
- 翻译或文本校正模型返回残缺 JSON 时自动重发，最多三次。
- 翻译结果逐句写入原子检查点；任务重试时复用源文本一致的已完成部分。
- `ensemble_audit.json` 记录每个复核窗口的三个候选、相似度和最终采用来源。
